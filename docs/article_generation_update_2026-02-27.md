# 📝 HueBrief AI 기사 생성 로직 및 UI/UX 통합 기획서 (Dual Mode)

**문서 개요**
본 문서는 HueBrief 플랫폼 내에서 AI를 활용한 기사 생성 시 적용되는 핵심 원칙과 UI/UX 플로우를 정의합니다. 기존의 과도한 '유사성 차단' 제약을 벗어나, 팩트를 유지하면서도 창의적인 텔링이 가능하도록 **[짧은 브리핑 모드]**와 **[인터랙티브 스토리보드 모드]**의 듀얼 트랙을 지원합니다.

---

## 1. 기사 생성 핵심 원칙 (Core Generation Rules)

1. **독립적 재구성 및 창의성 허용 (Creative Reconstruction)**
   * 크롤링해 온 외부 기사는 철저히 **참조(Reference) 및 팩트(Fact) 파악용**으로만 사용됩니다.
   * 표절을 막기 위해 원문의 문장 구조를 복사하는 것은 금지하되, **대화체, 해설톤, 스토리텔링 등 화자(Speaker)의 시점을 자유롭게 변환하는 창의적 윤색은 적극 허용**하여 기사의 풍부함을 살립니다.
2. **사실 무결성과 감정 연출의 분리 (Content Integrity)**
   * 선택된 감정(Hue)은 기사의 핵심 사실이나 수치를 절대 왜곡하지 않습니다.
   * 감정은 오직 **자극의 강도, 설명의 밀도, 표현의 톤(Tone) 및 시각적 연출 방식(인터랙티브 모드 시)**만을 조절하는 UX 레이어로 작동합니다.
3. **명확한 출처 보장 (Strict Source Grounding)**
   * 모든 생성 결과물은 유효한 크롤링 레퍼런스를 바탕으로 해야 하며, 결과물 데이터에 `sourceCitation`이 반드시 포함되어야 합니다.

---

## 2. 듀얼 모드 기사 생성 (Dual Generation Modes)

작성자는 기사 생성 전, 목적에 맞게 두 가지 포맷 중 하나를 선택합니다.

### ⚡ Mode A: 짧은 감성 브리핑 (Short Brief)
* **목적:** 바쁜 독자를 위해 핵심 정보만 빠르게 전달하면서도, 지정된 '감정 톤'으로 부드럽게 재구성된 브리핑.
* **AI의 역할:** 팩트 요약 및 감성적 윤색을 담당하는 **'감성 에디터'**.
* **특징:** 모바일 환경에서 한 화면에 들어오는 간결한 분량 (약 300~500자 내외).

### 🎬 Mode B: 인터랙티브 심층 기사 (Interactive Longform)
* **목적:** 독자가 스크롤하며 몰입할 수 있는 동적이고 시각적인 스토리텔링 경험 제공.
* **AI의 역할:** 웹페이지 섹션(Section)을 나누고, 시각적 연출(Trigger/Animation) 방향까지 제시하는 **'스토리보드(대본) 감독'**.
* **특징:** 스크롤 위치에 따른 텍스트 등장, 미디어 전환 등 입체적인 대본 블록 생성.

---

## 3. 기사 작성 UI/UX 프로세스 (Creation Flow)

### STEP 1: 레퍼런스 선택 및 모드 결정
* 사용자가 키워드 검색 후, 상위 기사 후보 중 참조할 기사의 팩트를 선택합니다.
* 화면 상단의 토글/버튼을 통해 **[짧은 브리핑]** 또는 **[인터랙티브 기사]** 생성 모드를 선택합니다.
* 적용할 **감정 카테고리(Hue)**를 선택합니다.

### STEP 2: 임팩트 있는 제목 추천 (AI Title Suggestions)
* **절대 규칙:** 원문 기사의 제목이나 구조를 그대로 재사용하지 않습니다.
* **UX 플로우:** AI가 본문 내용을 바탕으로 새롭게 재구성한 **짧고 임팩트 있는 제목 후보 3가지**를 상단에 칩(Chip) 형태로 추천합니다.
  * *구성 예: [직관적 팩트형] / [감성 터치형] / [호기심 유발형]*
* 사용자는 칩을 클릭하여 제목을 바로 적용하거나, 이를 바탕으로 직접 수정할 수 있습니다.

### STEP 3: 본문/대본 생성 및 감정 톤 적용
* **짧은 브리핑:** 선택한 감정 톤(예: '차분한 명료함')에 맞춰 텍스트의 어조와 단어 선택이 즉각 변환되어 에디터에 출력됩니다.
* **인터랙티브 기사:** 텍스트뿐만 아니라, 지정된 감정에 어울리는 화면 연출 지시어(예: "배경 서서히 어두워짐", "텍스트 페이드인")가 포함된 스크롤 섹션별 블록이 생성됩니다.

### STEP 4: 조건부 미디어 배치 UI (Conditional Media Placement)
* **UX 플로우:** 초기 작성 화면(빈 에디터 혹은 텍스트만 있는 상태)에서는 미디어(이미지/영상) 레이아웃이나 순서 조정 메뉴가 **노출되지 않습니다.**
* 사용자가 에디터 내에 미디어를 **'업로드'** 하거나 AI 이미지 **'생성'**을 완료하여 미디어가 화면에 추가된 직후에만, 해당 미디어 블록 근처에 **[위치 정렬 / 스크롤 섹션 지정 / 순서 변경] 툴바**가 부드럽게 나타납니다.
* 이를 통해 텍스트 본문(뼈대) 구성에 먼저 집중하게 한 뒤, 시각 자료 배치를 자연스럽게 유도하여 UI 복잡도를 낮춥니다.

### STEP 5: 명확한 출처 분리 표기 (Source Citation Display)
* **UX 플로우:** 출처를 기사 본문 마지막 문단에 텍스트로 뭉뚱그려 적지 않습니다.
* 기사 본문 영역이 완전히 끝난 하단에, 시각적으로 명확히 분리된(배경색이 다르거나 테두리가 있는) 별도의 컨테이너 UI를 제공합니다.
* 해당 영역의 타이틀은 **'📌 참고 출처'**로 고정 표기하며, 레퍼런스로 사용한 원문의 제목, 매체명, 링크가 자동으로 바인딩됩니다.

## 4. 모드별 AI 프롬프트 스키마 (개발 참조용 JSON)

개발팀은 사용자가 선택한 모드에 따라 아래와 같이 서로 다른 JSON 스키마를 호출하도록 분기 처리합니다. 프롬프트 지시어에 "단순 요약이 아닌 화자의 시점을 바꾼 창의적 서술"을 명시해야 합니다.

**[Mode A] 짧은 브리핑 기사 스키마**
```json
{
  "titles": ["후킹용 짧은 제목 1", "감성 터치 제목 2", "직관적 팩트 제목 3"],
  "emotion_tone": "serenity",
  "content_body": "원문을 바탕으로 감정 톤을 입혀 재구성한 500자 이내의 브리핑 텍스트",
  "sourceCitation": [
    {"title": "원문 기사 제목", "url": "https://...", "publisher": "매체명"}
  ]
}

**[Mode B] 인터랙티브 심층 기사 스키마**
```json
{
  "titles": ["스토리텔링형 제목 1", "임팩트 제목 2", "서사형 제목 3"],
  "emotion_tone": "immersion",
  "interactive_storyboard": [
    {
      "sequence": 1,
      "scroll_trigger": "0-30%",
      "visual_hint": "화면 전체를 덮는 어두운 톤의 이미지 배치 추천 (예: 폭우가 쏟아지는 도심)",
      "text_content": "도입부: 독자의 시선을 끄는 몰입형 문장"
    },
    {
      "sequence": 2,
      "scroll_trigger": "30-70%",
      "visual_hint": "데이터 차트 또는 팩트 강조 이미지",
      "text_content": "중반부: 핵심 팩트와 수치를 설명하는 해설 텍스트"
    }
  ],
  "sourceCitation": [
    {"title": "원문 기사 제목", "url": "https://...", "publisher": "매체명"}
  ]
}

## 5. 스마트 품질 관리 게이트 (Smart Quality Gate)

창의성 허용 기준 완화: 기존의 빡빡한 '문장 구조/단어 중복률 검사'를 완화하여 기계적인 생성 실패(Block)를 줄입니다.

팩트 체크 중심: 대신, 레퍼런스 기사에 없는 새로운 고유명사나 수치가 날조(Hallucination)되었는지를 중점적으로 검증합니다.

에러 발생 시 기사 전체를 초기화하지 않고, 수정이 필요한 특정 문단만 하이라이트하여 재생성을 유도합니다.
