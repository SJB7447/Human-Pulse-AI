## 9) Responsive Interaction Baseline (Validated)
> Epic 1~4 검증 결과를 반영한 운영 기준 (HueBrief 공통)

### 9.1 적용 범위 / 브레이크포인트
- Desktop: `>= 1280px`
- Tablet: `768px ~ 1279px`
- Mobile: `<= 767px`
- 반응형 전 구간에서 상태 전이(카드→상세→다음콘텐츠) 맥락이 유지되어야 한다.

### 9.2 Card → Detail 전환 원칙
- 카드 상세 진입 시 원 카드 맥락(위치/출처)을 유지한다.
- 복귀 시 리스트 문맥(스크롤/카드 위치)을 깨지 않는다.
- Hover 의존 패턴은 모바일/태블릿에서 강제하지 않는다.
- 키보드 진입(Enter/Space), focus-visible 상태를 기본 지원한다.

### 9.3 Scroll Text Rhythm 원칙
- 텍스트는 문장/문단 의미 단위로 노출한다.
- 과도한 단어 단위 모션은 금지한다.
- `prefers-reduced-motion` 환경에서 모션을 축소/제거한다.
- 작은 화면에서는 읽기 우선(가독성/행간/정렬)으로 구성한다.

### 9.4 Background Transition 원칙
- 배경 컬러 전환은 콘텐츠 이해를 돕는 보조 장치로만 사용한다.
- 스크롤 진행에 따라 자연스럽게 변하되, 본문 대비를 항상 유지한다.
- 빠른 스크롤에서도 점프/깜빡임이 없어야 한다.

### 9.5 Next Content Handoff 원칙
- 기사 말미 전환은 사용자 선택 기반으로 제공한다.
- 추천 콘텐츠는 현재 콘텐츠 하단에서 자연스럽게 이어지게 구성한다.
- 불필요한 중복 CTA는 지양하고 맥락 단절을 방지한다.

### 9.6 QA 게이트
- 필수 체크: `npm run dev`, `npm run lint`, `npm test`
- 배포 기준:
  - UX 체크리스트 90% 이상 통과
  - Blocker(맥락 손실/읽기 방해/복귀 실패) 0건
  - 성능 이슈 재현 시 원인 기록 및 최소 patch 제안
